== NAME

cz - conventional commit message builder

== SYNOPSIS

*cz* [*-r* | *--no-require-scope*] [*-d* | *--no-defined-scope*] [*-e* | *--no-enforce-patterns*] [*-m* | *--no-multi-scope*] [*-b* | *--no-breaking-footer*] [*-q*] [*-c* _FILE_] [_SUBCOMMAND_]

*cz* *create* +
*cz* *lint* [*-p* _PATH_]... +
*cz* *parse* +
*cz* *init* [*-o* _FILE_] [*-f*] +
*cz* *hook* [*install* | *uninstall* | *status*]

== DESCRIPTION

*cz* is an interactive tool for creating commit messages that follow the
Conventional Commits specification. It can also lint existing messages or
display the resolved configuration.

When invoked without a subcommand, *cz* defaults to *create* mode if run
in a terminal, or *lint* mode if receiving input on standard input.

== SUBCOMMANDS

*create*::
  Compose a commit message interactively using *gum*(1). Prompts for type,
  scope, breaking change indicator, description, body, and footer. Output
  is written to standard output. Default when run interactively.
  Requires *gum* to be installed.

*lint* [*-p* _PATH_]...::
  Validate a commit message against the Conventional Commits rules and the
  *.gitcommitizen*(5) configuration. Reads from standard input. Prints
  diagnostic messages explaining any errors. Exits non-zero if validation
  fails. Use *-e* to enforce scope matching against file patterns.
+
--
*-p*, *--paths* _PATH_:::
  Validate that the commit scope matches the specified file path. Can be
  repeated for multiple paths. Supports filenames with spaces. Also accepts
  newline-separated paths in a single argument (e.g., from `git diff
  --name-only`). Requires *-e* (enforce-patterns) to actually validate the
  scope matches.
--

*parse*::
  Display configuration with all values expanded. Useful for debugging.

*init* [*-o* _FILE_] [*-f*]::
  Generate a starter *.gitcommitizen* configuration. Prints to standard output
  by default, allowing piping or redirection.
+
--
*-o*, *--output* _FILE_:::
  Write to _FILE_ instead of stdout.

*-f*, *--force*:::
  Overwrite an existing file (only with *-o*).
--

*hook* *install* | *uninstall* | *status*::
  Manage the *commit-msg* git hook. *install* creates the hook to run
  *cz lint* on each commit. *uninstall* removes it. *status* shows whether
  the hook is installed.

== OPTIONS

*-c*, *--config-file* _FILE_::
  Use _FILE_ instead of *.gitcommitizen*. Searches parent directories if
  not specified.

*-r*, *--require-scope*::
  Require a scope to be present. For *create*: removes the "(none)" option.
  For *lint*: fails if commit has no scope. Overrides `require-scope` config.

*--no-require-scope*::
  Allow commits without scope. Overrides `require-scope` config.

*-d*, *--defined-scope*::
  Require scope to be defined in configuration. For *create*: removes the
  "(custom)" option, only allowing configured scopes. For *lint*: fails if
  scope is not in `[scopes]`. Overrides `defined-scope` config.

*--no-defined-scope*::
  Allow any scope, even if not in configuration. Overrides `defined-scope` config.

*-e*, *--enforce-patterns*::
  Enforce scope-file pattern matching. Requires scope to match file patterns
  from `[scopes]` when *--paths* is provided. Also implies *-d* (scope must
  exist in config). Overrides `enforce-patterns` config.

*--no-enforce-patterns*::
  Disable pattern enforcement. Overrides `enforce-patterns` config.

*-m*, *--multi-scope*::
  Allow multiple scopes like `feat(api,db):`. Overrides `multi-scope` config.

*--no-multi-scope*::
  Disallow multiple scopes. Overrides `multi-scope` config.

*-b*, *--breaking-footer*::
  Require a `BREAKING CHANGE:` (or `BREAKING-CHANGE:`) footer when the commit
  has a breaking change indicator (`!`). Overrides `breaking-footer` config.
  Default when unset: true.

*--no-breaking-footer*::
  Allow breaking changes without a `BREAKING CHANGE:` footer. Overrides
  `breaking-footer` config.

*-q*, *--quiet*::
  Suppress warnings and non-essential output.

*-h*, *--help*::
  Display usage information and exit.

*-V*, *--version*::
  Display version information and exit.

== FILES

*.gitcommitizen*::
  Project configuration file. See *gitcommitizen*(5) for format. If no
  configuration is found, the Angular convention is used (feat, fix, docs,
  style, refactor, perf, test, build, ci, chore, revert).

== BEHAVIOR TABLE

The *-r* and *-d* flags affect the *create* command:

[cols="1,3", options="header"]
|===
|Flags |create shows

|default (no flags)
|scopes + "(custom)" + "(none)"

|*-r* (require-scope)
|scopes + "(custom)" (scope mandatory, custom allowed)

|*-d* (defined-scope)
|scopes + "(none)" (can skip scope, no custom input)

|*-r -d* (both)
|scopes only (must pick configured scope)
|===

The *-r*, *-d*, and *-e* flags affect the *lint* command:

[cols="1,3", options="header"]
|===
|Flag |lint behavior

|default
|Validates format only; scope is optional

|*-r*
|Scope must be present

|*-d*
|Scope must exist in config (if provided)

|*-e*
|Scope must match file patterns (implies *-d*)

|*-r -e*
|Scope required and must match patterns

|*-b* / *--breaking-footer*
|Breaking `!` requires `BREAKING CHANGE:` or `BREAKING-CHANGE:` footer (default)

|*--no-breaking-footer*
|Breaking `!` allowed without footer
|===

CLI flags always override config settings.

== EXAMPLES

Create and commit interactively::

  git commit -m "$(cz)"

Create with required scopes (only configured scopes allowed)::

  git commit -m "$(cz -r -d)"

Lint the last commit message::

  git log -1 --pretty=%B | cz lint

Lint before committing (commit-msg hook)::

  cz lint < "$1"

Lint silently (exit status only)::

  cz lint -q < "$1" || exit 1

Lint with scope validation against specific files::

  echo "feat(api): add endpoint" | cz lint --paths "src/api/routes.ts"

Lint with scope-file pattern enforcement (in commit-msg hook)::

  files=$(git diff --cached --name-only)
  cz -e lint --paths "$files" < "$1"

Lint with multi-scope enabled::

  echo "feat(api,db): cross-cutting change" | cz -m lint

Show resolved configuration::

  cz parse

Generate a starter configuration::

  cz init

Install the commit-msg hook::

  cz hook install

Validate PR titles in GitHub Actions::
+
[source,yaml]
----
- uses: cachix/install-nix-action@v30

- name: Pre-build cz
  run: nix build --no-link github:proventuslabs/scriptorium#cz

- name: Validate PR title
  shell: nix shell github:proventuslabs/scriptorium#cz -c bash -e {0}
  env:
    GH_TOKEN: ${{ github.token }}
    PR_TITLE: ${{ github.event.pull_request.title }}
    PR_NUMBER: ${{ github.event.number }}
  run: |
    FILES=$(gh pr view "$PR_NUMBER" --json files -q '.files[].path')
    if ! OUTPUT=$(echo "$PR_TITLE" | cz lint --paths "$FILES" 2>&1); then
      echo "::error title=Invalid PR Title::$OUTPUT"
      exit 1
    fi
----

== EXIT STATUS

*0*::
  Success.

*1*::
  Runtime error (lint validation failed, missing dependency, config not found,
  not a git repository).

*2*::
  Usage error (invalid arguments).

*130*::
  User cancelled (SIGINT).
