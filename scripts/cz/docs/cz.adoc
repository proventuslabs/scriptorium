== NAME

cz - conventional commit message builder

== SYNOPSIS

*cz* [*-r* | *--no-require-scope*] [*-m* | *--no-multi-scope*] [*-q*] [*-c* _FILE_] [_SUBCOMMAND_]

*cz* *create* [*--custom-scope* | *--no-custom-scope*] +
*cz* *lint* [*-p* _PATHS_] +
*cz* *parse* +
*cz* *init* [*-f*] +
*cz* *hook* [*install* | *uninstall* | *status*]

== DESCRIPTION

*cz* is an interactive tool for creating commit messages that follow the
Conventional Commits specification. It can also lint existing messages or
display the resolved configuration.

When invoked without a subcommand, *cz* defaults to *create* mode if run
in a terminal, or *lint* mode if receiving input on standard input.

== SUBCOMMANDS

*create* [*--custom-scope* | *--no-custom-scope*]::
  Compose a commit message interactively using *gum*(1). Prompts for type,
  scope, breaking change indicator, description, body, and footer. Output
  is written to standard output. Default when run interactively.
  Requires *gum* to be installed.
+
--
*--custom-scope*:::
  Allow free-form scope input (default behavior).

*--no-custom-scope*:::
  Only allow scopes defined in configuration (no free-form input).
  The "(none)" option is still available unless *--require-scope* is also set.
--

*lint* [*-p* _PATHS_]::
  Validate a commit message against the Conventional Commits rules and the
  *.gitcommitizen*(5) configuration. Reads from standard input. Prints
  diagnostic messages explaining any errors. Exits non-zero if validation
  fails. When *--require-scope* is enabled, validates that scopes are defined
  in configuration and requires a scope when files match scope patterns.
+
--
*-p*, *--paths* _PATHS_:::
  Validate that the commit scope matches the specified file paths. Accepts a
  space-separated list of paths. Uses glob patterns from the `[scopes]`
  section of the configuration to verify the scope is appropriate for the
  files being committed.
--

*parse*::
  Display configuration with all values expanded. Useful for debugging.

*init* [*-o* _FILE_] [*-f*]::
  Generate a starter *.gitcommitizen* configuration. Prints to standard output
  by default, allowing piping or redirection.
+
--
*-o*, *--output* _FILE_:::
  Write to _FILE_ instead of stdout.

*-f*, *--force*:::
  Overwrite an existing file (only with *-o*).
--

*hook* *install* | *uninstall* | *status*::
  Manage the *commit-msg* git hook. *install* creates the hook to run
  *cz lint* on each commit. *uninstall* removes it. *status* shows whether
  the hook is installed.

== OPTIONS

*-c*, *--config-file* _FILE_::
  Use _FILE_ instead of *.gitcommitizen*. Searches parent directories if
  not specified.

*-r*, *--require-scope*::
  Enforce scope requirements. Affects both *create* and *lint* commands.
  For *create*: scope is mandatory (no "(none)" option). Custom input is still
  allowed unless *--no-custom-scope* is also set.
  For *lint*: requires scope when files match scope patterns, and validates
  that any scope used is defined in configuration. Overrides the `require-scope`
  setting in the configuration file. See BEHAVIOR TABLE below for details.

*--no-require-scope*::
  Disable scope requirements. Allows custom scopes in *create* and commits
  without scope in *lint* even when files match scope patterns. Overrides
  the `require-scope` setting in the configuration file.

*-m*, *--multi-scope*::
  Allow multiple scopes in a commit like `feat(api,db):`. Overrides the
  `multi-scope` setting in the configuration file.

*--no-multi-scope*::
  Disallow multiple scopes. Commits with multiple scopes like `feat(api,db):`
  will fail validation. Overrides the `multi-scope` setting in the
  configuration file.

*-q*, *--quiet*::
  Suppress warnings and non-essential output.

*-h*, *--help*::
  Display usage information and exit.

*-V*, *--version*::
  Display version information and exit.

== FILES

*.gitcommitizen*::
  Project configuration file. See *gitcommitizen*(5) for format. If no
  configuration is found, the Angular convention is used (feat, fix, docs,
  style, refactor, perf, test, build, ci, chore, revert).

== BEHAVIOR TABLE

The *--require-scope* and *--no-custom-scope* options affect the *create* command:

[cols="1,3", options="header"]
|===
|Flags |create shows

|default (no flags)
|scopes + "(custom)" + "(none)"

|*--require-scope*
|scopes + "(custom)" (scope mandatory, custom allowed)

|*--no-custom-scope*
|scopes + "(none)" (can skip scope, no custom input)

|*--require-scope --no-custom-scope*
|scopes only (must pick configured scope)
|===

The *--require-scope* option also affects the *lint* command:

[cols="1,3", options="header"]
|===
|Flag |lint behavior

|default
|Validates format; scope is optional

|*--require-scope*
|Requires scope for scoped files; validates scope exists in config

|*--no-require-scope*
|Allows no scope even for scoped files
|===

The config settings `require-scope = true` and `multi-scope = true` in `[settings]`
have the same effect as their corresponding CLI flags, but CLI flags always
override config settings.

== EXAMPLES

Create and commit interactively::

  git commit -m "$(cz)"

Create with required scopes (only configured scopes allowed)::

  git commit -m "$(cz --require-scope)"

Lint the last commit message::

  git log -1 --pretty=%B | cz lint

Lint before committing (commit-msg hook)::

  cz lint < "$1"

Lint silently (exit status only)::

  cz lint -q < "$1" || exit 1

Lint with scope validation against specific files::

  echo "feat(api): add endpoint" | cz lint --paths "src/api/routes.ts"

Lint with required scope for scoped files::

  cz --require-scope lint --paths "$(git diff --cached --name-only)" < "$1"

Lint with multi-scope enabled::

  echo "feat(api,db): cross-cutting change" | cz --multi-scope lint

Show resolved configuration::

  cz parse

Generate a starter configuration::

  cz init

Install the commit-msg hook::

  cz hook install

== EXIT STATUS

*0*::
  Success.

*1*::
  Runtime error (lint validation failed, missing dependency, config not found,
  not a git repository).

*2*::
  Usage error (invalid arguments).

*130*::
  User cancelled (SIGINT).
